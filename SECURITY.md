# Security Policy

## Reporting a Vulnerability
Please report security issues privately to the repository maintainers.

When reporting, include:
- Affected component/file
- Reproduction steps
- Potential impact
- Suggested mitigation (if known)

Do not open public issues for unpatched vulnerabilities.

## Secret Handling
- Never commit credentials, API keys, or private tokens.
- Use environment variables and untracked local files.
- Rotate secrets immediately if they are exposed.
